## 2026-07-18 — deferred-work cycle triage (spec-deferred-work-cycle)

Triage of all 9 items above, run per `_bmad-output/implementation-artifacts/spec-deferred-work-cycle.md`. Every item now has an explicit disposition; no existing entry above was edited.

1. **PROTOCOL_VERSION join gate** (spec-0-1, item 1) — RESOLVED by Story 0.2. Client sends `pv` in join options (`client/src/net/connection.ts`); server rejects mismatches in `ArenaRoom.onAuth` via `ServerError(ErrorCode.AUTH_FAILED = 525)` built by `protocolVersionError()` (`server/src/rooms/roomOptions.ts`); client maps code 525 to "VERSION MISMATCH — PLEASE REFRESH THE PAGE" (`connectErrorStatus`). Covered by `client/src/__tests__/connection.test.ts` and `server/src/__tests__/reconnect.test.ts`.
2. **JOINING-state client guard** (spec-0-1, item 2) — RESOLVED by Story 0.3. `afterStep` skips frame sends to any client whose state isn't `JOINED`; `armJoiningDeadline`/`kickIfStillJoining` (using `CONFIG.net.joiningDeadlineSeconds = 10`) kicks clients stuck in JOINING and is re-armed on reconnect. Had zero regression coverage before this cycle; added `server/src/__tests__/joiningGuard.test.ts` this cycle to pin the frame-guard skip and the fire-time kick decision (deadline scheduling and the reconnect re-arm remain verified by inspection, not tests).
3. **Transport force-disconnect decision** (spec-0-1, item 3) — CLOSED. Its recorded revisit condition ("alongside the PROTOCOL_VERSION join gate") is now satisfied — stale clients get a clean 525 rejection at matchmake time, before the transport ever sees them. The accepted behavior (Colyseus 0.17 transport-kicks malformed/unknown messages rather than log-and-drop) stands unchanged.
4. **matchSmoke reliability** (2026-07-17 section) — FIXED THIS CYCLE, scope confined to `server/scripts/matchSmoke.mjs`. The 90u drone-camp pocket became a ~13.5u floor (`endRadiusFraction: 0.015`) that only the scripted human can hold — drone AI never throttles below 0.5 so it can't loiter there and storm-sinks, while A parks inside (a pure 0 floor is an hp race A's 100hp cruiser can't reliably win against the 120hp battleship drone). Torpedo-luck fixed at the delivery mechanics, not just budgets: proactive island avoidance (deterministic `generateMap` from the welcome seed), LOS-gated firing via shared `segCircleHit`, racetrack strafing vs a latched-stationary target, astern unstick. Step-5 assertions relaxed to race-independent lifecycle invariants (winner A placed 1st, every hull placed ≥1, B bounded 2..6). Plus `portOpen` pre-boot guard + SIGKILL teardown fallback, and wider budgets. Proven by 3 consecutive clean full-lifecycle runs (~5.5 min each). No production code touched.
